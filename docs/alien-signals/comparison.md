# 方案对比（要点）

对比维度：依赖精度、热路径开销、调度策略、抽象厚度、生态集成。

- alien-signals
  - 依赖精度：值级别（signal 槽）
  - 热路径：单形对象 + 版本检查 + 去重队列
  - 调度：微任务/小队列 + 拓扑安全
  - 抽象：薄；仅 signal/computed/effect
  - 集成：需另行绑定到视图（更灵活）
- Vue 3（reactivity）
  - 依赖精度：属性级（Proxy）
  - 热路径：trap + effect track/trigger（通用）
  - 调度：组件优先 + 异步队列
  - 抽象：厚；与组件/渲染器深结合
  - 集成：一体化，易用
- Solid / @preact/signals
  - 依赖精度：值/表达式
  - 热路径：版本 + 脏标 + 去重
  - 调度：与渲染绑定良好
  - 抽象：薄（Solid）/更薄（Preact signals）
  - 集成：与各自生态匹配
- RxJS
  - 依赖精度：流（push-only）
  - 热路径：订阅/运算符链（更复杂）
  - 调度：多种 scheduler/背压等
  - 抽象：厚；功能强大但非 UI 专用
  - 集成：跨平台/通用事件流

结论：若目标是“在 JS 中获取极致的本地响应性能”，信号 + 版本/脏位 + 去重调度的轻实现路线（alien-signals）是更优选；若需要组件/模板/生态一体化，成熟框架的响应系统仍更省心。
