# 方案对比（要点）

原生小程序语法

- 最低抽象，能力完整；但 TS 友好性不足，逻辑复用与复杂页面的代码组织成本较高。
- wevu：在原生之上做增强，组合式 API 改善状态与副作用管理；可与原生并存。

编译时方案（如早期类 React/JSX -> 原生模板）

- JSX/Render Function 的动态性与原生模板的静态性存在天然错配，导致“反向还原”难题。
- wevu：无编译时转换，避免编译限制带来的不可解边界。

运行时 VDOM 方案（如运行时模拟 DOM 或自定义 reconciler）

- 大运行时体积 + 递归 VDOM 渲染，复杂页面容易卡顿，Hello World 体积也相对较大。
- wevu：直接驱动小程序数据层，避免额外 VDOM 和抽象开销。

与其他增强型框架

- 某些方案基于 Vue 2 或使用“影子 Vue 组件”同步到小程序组件；抽象更厚，内存与调度开销更高。
- wevu：没有“影子组件”，直接在小程序组件上工作，抽象更少、可预测性更好。
