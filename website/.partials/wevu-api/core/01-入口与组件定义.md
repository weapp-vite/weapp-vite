## 入口与组件定义

### `createApp()` {#createapp}

<!-- api-reference-details -->

**类型签名：** `typeof import('wevu')['createApp']`

**运行时说明：** 该 API 运行在 Wevu 的组件作用域内；涉及 hook 或实例上下文时，应在同步 `setup()` 中调用。

**Vue 差异：** Vue Web 创建并挂载 DOM 应用；Wevu 创建的是小程序 App 注册描述，不接收根 DOM 容器，通常每个 App 入口只调用一次。

**示例：** 见 [入口与 Vue 兼容共用示例](/wevu/api/core#core-examples)。

- 类型入口：`CreateAppOptions` / `RuntimeApp`
- 用途：创建并注册小程序 App 运行时入口。
- 说明：通常在 App 侧只调用一次。

### `defineComponent()` {#definecomponent}

<!-- api-reference-details -->

**类型签名：** `typeof import('wevu')['defineComponent']`

**运行时说明：** 该 API 运行在 Wevu 的组件作用域内；涉及 hook 或实例上下文时，应在同步 `setup()` 中调用。

**示例：** 见 [入口与 Vue 兼容共用示例](/wevu/api/core#core-examples)。

- 类型入口：`DefineComponentOptions` / `ComponentDefinition`
- 用途：定义页面/组件并接入 Wevu 生命周期与响应式。
- 说明：Wevu 统一使用 `Component()` 模型。
